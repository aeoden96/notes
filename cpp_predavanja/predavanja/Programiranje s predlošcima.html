SFINAE (substitution-failure-is-not-an-error)

SFINAE je princip koji omoguƒáava preoptereƒáenje predlo≈æaka funkcija ali ujedno i neke specifiƒçne tehnike programiranja. Princip se sastoji u sljedeƒáem:

Primjena: identificiramo da li zadani tip ima podtip X.

Zatim definiramo dva predlo≈°ka funkcije: prvi uzima pokazivaƒç na konstantan podtip X dok drugi uzima sve vrste argumenta (...)‚Äâ‚Äî‚Äâto je C-verzija funkcije s proizvoljnim brojem argumenataza kori≈°tenje vidi www.cplusplus.com/reference/cstdarg/.

template <typename T> RT2 test(...);

Definirajmo jedan tip koji ima podtip X.

    typedef int X;
   // ..
};

int main()
{

    std::cout << sizeof(test<double>(0)) << std::endl;  // ispisuje 2

Kod funkcionira na sljedeƒái naƒçin:

  • prevodilac treba odabrati koji ƒáe funkcijski predlo≈æak instancirati na osnovu parametra (0) koji je tipa const int.

  • Takav se parametar mo≈æe konvertirati u pokazivaƒç na const T::X (naravno kao tip T ima podtip X).

  • Ta konverzija je slabija od konverzije koja je potrebna za funkciju koja uzima proizvoljne parametre (...).

To znaƒçi da ƒáe prevodilac preferirati test koji vraƒáa RT1 ukoliko T ima podtip X.

Time smo dobili tehniku kojom mo≈æemo provjeriti (za vrijeme kompilacije) da li neki tip ima podtip X (sizeof daje 1) ili nema (sizeof daje 2). Kao sljedeƒái korak mo≈æemo definirati makro:

Numeriƒçki kod vrlo ƒçesto radi s petljama u kojima se raƒçunaju elementarne operacije nad vektorima. Na primjer, skalarni produkt bismo prirodno implementirali na ovaj naƒçin:
