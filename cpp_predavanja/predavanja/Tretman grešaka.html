Tretman grešaka

Mladen Jurak
version 2.0, 2018-02

Izuzeci

  • Sustav izuzetaka separira detekciju greške od reagiranja na grešku. Dio koda koji detektira grešku izbacuje izuzetak, a dio koji reagira na grešku zatvoren je u odgovarajući catch blok. Komunikacija između dva dijela programa vrši se kroz izbačeni objekt.

  • Kod koji izbacuje izuzetak pomoću throw naredbe zatvara se u try blok. Tip izbačenog objekta određuje koji će kod za obradu greške (error handler) biti pozvan. Pravilo je da će biti pozvan najbliži catch blok u pozivnom lancu koji odgovara tipu izbačenog objekta. 

try{
    // kod koji mpže izbaciti izuzetak
}
catch(TipIzuzetka1 e){
    // kod koji procesira izuzetak tipa TipIzuzetka1
}
catch(TipIzuzetka2 const & e){
    // kod koji procesira izuzetak tipa TipIzuzetka2
}

// Ovdje se nastavlja kod ako je izuzetak
// izbačen ali i ako nije izbačen

  • catch blokova može biti proizvoljno mnogo.

try—catch

  • Kod za obradu greške zatvara se u catch blok koji deklarira izuzetak koji hvata slično kao što funkcija deklarira svoj parametar. Izuzeci se izbacuju i hvataju slično kao što se argumenti predaju funkciji. Izuzetak može biti bilo kojeg tipa s ograničenjem da tip mora dozvoljavati kopiranje.

  • Kada se izvrši throw naredba sekvencijalno izvršavanje programa se prekida i počinje potraga za odgovarajućim catch blokom. Ako se throw naredba nalazi unutar nekog try bloka, onda se pretražuju njemu pridruženi catch blokovi. Ako je blok nađen izuzetak će biti tretiran u nađenom catch bloku i program se nastavlja iza posljednjeg catch bloka pridruženog try bloku u kome je izbačen izuzetak. 

#include <stdexcept> // za bad_alloc
#include <cstdlib>   // za exit
#include <iostream>

int main(){

    double * ax = nullptr;
    double * px = nullptr;
    try{
        ax = new double[7];
        unsigned long size = 10000000000;
        px = new double[size];                           // izbacuje izuzetak std::bad_alloc
                                                         // sljedeća linja koda više neće biti izvršena
        for(unsigned int i=0; i< size; ++i) px[i] = 2*i-1;
        // ... procesiramo px i ax
        delete [] px;  // dealokacija
        delete [] ax;
    }
    catch(std::bad_alloc const & e){
        // Ispiši poruku o grešci.
        std::cout << "Uhvaćena je greška " << e.what() << std::endl;
        if(ax != nullptr) delete [] ax;   // ova memorija nije dealocirana
        std::exit(1);                     // završetak s greškom
    }

    return 0;
}

Prilikom traženja koda koji može procesirati grešku catch blok se selektira na osnovi varijable koju deklarira. Ime varijable se pri tome može ispustiti ako se izbačeni objekt ne koristi u bloku.

Podudaranje izbačenog objekta i tipa deklariranog u catch bloku ne mora biti egzaktno, ali je broj implicitnih konverzija sužen u odnosu na funkcijski poziv. Dozvoljene su sljedeće konverzije:

  • Konverzija iz nekonstantnog objekta u konstantan je dozvoljena;

  • Konverzija iz izvedenog u bazni tip je dozvoljena;

  • Polja i funkcije se konvertiraju u odgovarajuće pokazivače.

Druge konverzije nisu dozvoljene (standardne konverzije i slično).

Pri ulazu u catch blok njegov parametar se inicijalizira izbačenim objektom. Pri tome vrijede pravila kao i kod prijenosa argumenata funkciji. Ako catch blok deklarira parametar kao referencu na tip, onda će referenca biti samo inicijalizirana izbačenim objektom. Ako pak deklarira objekt, onda će doći do kopiranja izbačenog objekta u catch blok i catch blok će raditi s kopijom izbačenog objekta. Sam izbačeni objekt tada je nedostupan u catch bloku.

Izbacivanje izuzetaka

Objekt se izbacuje pomoću throw naredbe. Iza throw dolazi konstruktor objekta kojeg izbacujemo.

#include <string>
#include <stdexcept>  // za runtime_error
#include <iostream>
#include <fstream>    // za ifstream

class FileProcessor{
   // ...
};

void foo(std::string const & file_name)
{
    FileProcessor proc;   // lokalni objekt
    double * px = new double; // dinamički alocirani loc objekt
    std::ifstream in(file_name);
    // Izbaci izuzetak ako file nije otvoren
    if( !in.is_open() )
        throw std::runtime_error("Cannot open file "+file_name);
    //...
    delete px;
}

int main()
{
    try{
        foo("example.dat");
    }
    catch(std::runtime_error const & e){
        std::cout << "Error: " << e.what() << std::endl;
    }
    // Program se nastavlja ovdje
    // ...
}

  • Ako se throw ne nalazi u try bloku (kao u funkciji foo), onda se izlazi iz funkcije u kojoj je izuzetak izbačen i uništavaju se lokalni objekti (ovdje proc). Ako je funkcija smještena u jedan try blok potraga se nastavlja u njoj pripadajućim catch blokovima. Ako je catch blok nađen, njegov kod obrađuje izuzetak i program se nastavlja iz poslijednjeg catch bloka pridruženog try bloku u kome je poziv funkcije. U suprotnom se potraga nastavlja u pozivnoj funkciji, a program se završava ako catch blok na kraju nije nađen. Ova se procedura naziva stack unwinding.

Činjenica da catch blok može biti izvan lokalnog dosega u kome se poziva throw ima za posljedicu:

  • Izvršavanje lokalnog bloka (funkcije) se prekida i sve lokalne varijable izlaze iz dosega i pri tome se uništavaju. Na svim lokalnim objektima se poziva njihov destruktor dok dinamički alocirana memorija ostaje nedealocirana.

STL izuzeci

Standardna biblioteka definira jedan broj klasa koje predstavljaju izuzetke u zaglavlju &lt;stddexcept&gt;.

  • Svi ti izuzeci proširuju klasu std:exception.

  • Namijenjeni su podršci STL klasama (new izbacuje bad_alloc, itd) no može ih koristiti i korisnički kod.

Višestruki catch blokovi

  • Možemo imati više catch blokova ako svaki deklarira različit argument.

  • catch blokovi se procesiraju sekvencijalno u redosljedu u kojem su napisani.

  • Ako su argumenti catch blokova vezani odnosima nasljeđivanja, onda prvo navodimo blokove koji hvataju specijaliziranije argumente.

  • Kao zadnji catch blok možemo staviti blok koji hvata sve moguće izuzetke (...). 

#include <stdexcept>
#include <iostream>

int main()
{

    try{
        // kod koji može izbaciti runtime_error ili
        // općenitiji exception
        throw std::invalid_argument("Expecting ... but ... !");
    }
    catch(std::runtime_error const & e){
        std::cout << "Izbačen je runtime_error! " << e.what() << std::endl;
    }
    catch(std::exception const & e){ // procesuiramo širu klasu izuzetaka
        std::cout << "Izbačen je exception!" << e.what() << std::endl;
    }
    catch(...){ // Ova konstrukcija prihvaća svaki izuzetak
        std::cout << "Izbačen je nepoznati izuzetak!" << std::endl;
    }
    return 0;
}

Kada catch blok hvata objekte bazne klase, on može uhvatiti i objekte izvedene klase. Pri tome polimorfno ponašanje ovisi o tome da li catch blok deklarira svoj parametar kao referencu ili kao objekt.

Napomene

  • Destruktor ne bi smio nikada izbaciti izuzetak. Naime, destruktori se pozivaju kod izbacivanja izuzetka radi destrukcije lokanih varijabli kada je catch blok u širem dosegu (npr. u pozivnom programu). Kada destruktor u takvoj situaciji izbaci izuzetak postaje nejasno koji od ta dva izuzetka treba imati prednost. Tada će biti pozvana funkcija terminate iz standardne biblioteke koja zove abort koja zaustavlja program. Stoga je dobra praksa sve moguće izuzetke obraditi unutar destruktora. To nije naročit problem jer destruktor samo dealocira resurse, što je operacija koja rijetko izbacuje izuzetke.

  • S druge strane, konstruktor se može prirodno naći u situaciji da vrši operacije koje mogu izbaciti izuzetak. Tada se pojavljuje opasnost da objekt neće biti korektno konstruiran što treba izbjeći procesiranjem izuzetka. No i u situaciji kad se to desi imamo garanciju da će konstruirane varijable/objekti biti propisno uništeni.

  • Ponekad kod u catch bloku može tek djelomično procesirati nastalu grešku. Tada on može ponovo izbaciti izuzetak pomoću throw. Ovdje iza throw ne dolazi objekt koji se izbacuje. Takva konstrukcija je dozvoljena samo u catch bloku.

  • U složenijim programima ponekad pišemo vlastitu hierarhiju klasa izuzetaka.

Funkcijski try blok

Izuzetak može biti izbačen u inicijalizacijskoj listi konstruktora i tada on može biti uhvaćen tek posebnom sintaksom prikazanom niže. Ključna riječ try se stavlja prije inicijalizacijske liste, a catch blok se nastavlja odmah iza tijela konstruktora.

template <typename T>
class C{
    public:
        C(long int);
        int size() const { return size_;}
        // ... etc
    private:
        T* pT;
        int size_;
};

template <typename T>
C<T>::C(long int s) try : pT(new T[s]), size_(s)
{
    // prazno tijelo
}catch(const std::bad_alloc& e)
 { cout << e.what() << endl; }

To je jedini način da se uhvati izuzetak izbačen u inicijalizacijskoj listi.

Specifikacija izuzetaka

Moguće je pri deklaraciji funkcije deklarirati da funkcija ne izbacuje izuzetke pomoći noexcept:

void f(int x) noexcept
{
    // ..
}

Prevodilac ne provjerava za vrijeme kompilacije da li noexcept funkcija zaista ne izbacuje izuzetke, te stoga u noexcept funkciji možemo imati i throw naredbu. Ako izuzetak bude izbačen program završava pozivom terminate funkciji.

noexcept uzima eventualni dodatni argument true ili false sa značenjem true = ne izbacuje izuzetak, false = izbacuje:

int f() noexcept(true) {   // ne izbacuje izuzetke
    // ...
}
int g() noexcept(false) {   // može izbaciti izuzetak
    // ...
}

Postoji još i noexcept operator koji uzima izraz i vraća bool vrijednost koja indicira da li izraz može izbaciti izuzetak (false) ili ne može (true). Taj se operator koristi kako bi se uvjetno odredilo da li funkcija izbacuje ili ne.

int g() noexcept( noexcept(f()) ) {   // ne izbacuje ako f() ne izbacuje
    // ...
}

noexcept operator ne izračunava svoj argument već samo provjerava deklaracije funkcija koje izraz može pozvati i da ne sadrže throw.

Primjer

Imamo klasu FileReader koja čita varijable i njihove vrijednosti iz datoteke koja ima sljedeću formu:

   apscisa = 11.6
ordinata=777
 sss    = blabla

Svaka linija ima oblik ime_varijable = vrijednost. Bjeline su proizvoljne, a o tipovima varijabli nemamo pretpostavki (mogu biti svih C++ tipova). Klasa pamti i imena varijabli i njihove vrijednosti i dozvoljava traženje vrijednosti prema imenu.

// Klasa FileReader čita strukturiranu datoteku i nudi metodu get()
// pomoću koje se čita vrijednost varijable. Jedini izuzetak koji se izbacuje je
// std::runtime_error.
class FileReader{
  public:
    // Počitaj file i ispuni mapu variables. Izbacuje izuzetak ako file ne postoji,
    // nemamo prava čitanja ili file nije strukturiran u obliku
    // varijabla1 = vrijednost1
    // varijabla2 = vrijednost2
    // .....
    void readFile(std::string const & file_name);
    // Pronađi varijablu name i vrati njenu vrijednost tipa T. Izbacuje izuzetak
    // ako nema varijable name i ako se vrijednost ne može konvertirati u tip T
    template <typename T>
    T get(std::string const & name);
  private:
    // Vrijednosti varijabli pročitanih iz datoteke.
    // Prvi argument je ime varijable, a drugi njena vrijednost u obliku stringa
    std::map<std::string, std::string> variables;
    // implementacija -- pročitaj jednu liniju u datoteci i napravi par: ime varijable,
    // vrijednost varijable.
    std::pair<const std::string, std::string> readline(std::string const & line);
};

Primjer - glavni program

U glavnom programu pokušavamo učitavati varijable iz datoteke u try bloku. Ukoliko to ne uspije varijable inicijaliziramo defaultnim vrijednostima.

int main(int argc, char * argv[])
{
    std::string name{"default_name"};
    if(argc > 1) name = argv[1];

    FileReader fr;  // FileReader objekt
    double x = 0.0;
    int y = 0.0;
    try{
       fr.readFile(name);                // pročitaj ulaznu datoteku
       x = fr.get<double>("apscisa");    // čitanje varijabli
       y = fr.get<int>("ordinata");
    }
    catch(std::runtime_error const & e)    // ako je izbačena greška
    {                                      // nastavi ovdje
      std::cout << "Error : " << e.what() << std::endl;
      std::cout << "Biram defaultne vrijednosti za parametre x i y!\n";
      x = 15.4;
      y = 17;
    }
    // program nastavlja ovdje (sa i bez izbačenog izuzetka)
    // ...
    return 0;
}
