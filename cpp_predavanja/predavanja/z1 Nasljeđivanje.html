Implementacija matrice: klasa FullMatrix

  • Matricu pamtimo kao jedano polje pri čemu su elementi matrice spremljeni u to polje po stupcima (FORTRANSKI zapis). Pored toga pamtimo broj redaka i broj stupaca matrice.

  • m_data = pokazivač na polje elemenata, m_col; m_row broj stupaca i broj redaka.

  • Elemente matrice dohvaćamo sintaksom A(i,j), gdje indeksi idu od nule, i pri tome dohvaćamo element polja m_data[i + j*m_row ].

Napomene o klasi FullMatrix

  • Za konstrukciju matrice trebamo znati samo dimenzije matrice.

  • Matrica može biti pravokutna i kvadratna.

  • Defaultni konstruktor ne alocira memoriju.

  • Matrici možemo dinamički promijeniti dimenziju pomoću metode resize(). Ako povećavamo dimenziju matrice stare vrijednosti ostaju zapamćene; u suprotnom svi se elementi redimenzionirane matrice stavljaju na nulu.

  • Kontrola kopiranja (duboko kopiranje) mora biti implementirana.

  • Matrica ima privatnu funkciju koja vraća pokazivač na polje elemenata. Namijenjena je za korištenje isključivo u friend funkcijama.

  • row() daje broj redaka matrice, a col() daje broj stupaca.

Nova implementacija matrice: trakasta matrica (klasa BandMatrix)

Matrica je trakasta ako su svi njeni elementi različiti od nule smješteni na nekoliko dijagonala matrice. Broj dijagonala na kojima su elementi različiti od nule treba biti bitno manji od dimenzije matrice da bismo koristili trakastu matricu umjesto obične (full) matrice.

U trakastoj formi pamte se dijagonala matrice u kojima su elementi različiti od nule. Svi ostali elementi se ne pamte radi uštede prostora.

Trakasta matrica se zadaje svojom dimenzijom m_col (matrica je kvadratna), brojem poddijagonala iznad glavne dijagonale m_up i brojem poddijagonala ispod glavne dijagonale m_low.

Ukupan broj dijagonala koje se pamte je 1 + m_up +m_low. Dijagonale se pamte kao reci matrice, tako da se umjesto matrice reda m_col x m_col pamti matrica reda (1 + m_up +m_low) x m_col. Element A(i,j) matrice nalazi se stoga na mjestu (i - j + m_up, j) u trakastoj matrici.

Elementi matrice se pamte u jednom vektoru (m_data) poredani po stupcima. Element A(i,j) stoga nalazimo na mjestu m_data[ i - j + m_up + j*(1+m_up+m_low) ].

U trakastoj matrici ne možemo mijenjati elemente izvan dijagonala koje se pamte. Stoga operator dohvata treba izbacivati izuzetak kada se ti elementi pokušavaju mijenjati. Ako se samo dohvaćaju (čitaju) onda operator dohvata treba vratiti nulu.

Klasa BandMatrix

Slična je klasi FullMatrix. Razlike: Metode upper_diag() i lower_diag() koje vraćaju broj naddiagonala i poddiagonala. Metoda resize uzima drugačije argumente, kao i konstruktori. Dodali smo metodu print() koja ispisuje matricu u trakastoj formi.

Apstraktna bazna klasa

Da bismo različite tipove matrica mogi koristiti ne razlikujući im tip napravimo im zajedničku bazu.

Napomeni o apstraktnoj bazi

  • Bazna klasa omogućava uniformno tretiranje FullMatrix i BandMatrix objekata kroz pokazivače i reference tipa Matrix.

  • Bazna klasa ne alocira memoriju pa stoga većinu metoda ne možemo implementirati; ona je stoga apstraktna. Sastoji se uglavnom od čistih virtualnih funkcija.

  • Funkcija clear() postavlja sve elemente matrice na nulu. Uočimo da ona ima implementaciju jer se može implementirati pomoću metoda iz klase Matrix. Izvedene klase nasljeđuju tu metodu i ne moraju je posebno implementirati.

  • Operator ispisa je prijatelj klase Matrix i za svoju implementaciju treba samo sučelje klase Matrix. Time dobivamo operator koji može ispisati svaku matricu koja implementira sučelje Matrix. BandMatrix ima svoju metodu print() no ona ispisuje matricu onako kako je implementirana, dok operator ispisa u baznoj klasi daje standardni ispis matrice.

Sustavi linearnih algebarskih jednadžbi

Sustav linearnih algebarskih jednadžbi je objekt koji se sastoji od:

  • Matrice (mat),

  • Vektora desne strane (rhs),

  • Vektora rješenja (sol).

Osnovne metode su mu

  • assembly() koja formira matricu i vektor desne strane,

  • solve() koja riješava sustav.

Linearni sustav je kompozicija matrice i dva vektora. On kontrolira životnih vijek svojih sastavnih dijelova, koji ne postoje nezavisno od njega.

Sustav koji ima matricu u obliki FullMatrix nazvat ćemo FullSystem, a onaj koji koristi BandMatrix zovemo BandSystem. Prirodno je organizirati različite sustave u hierarhiju kako bismo ih mogli koristiti polimorfno.

System - klase

Napomene o stablu nasljeđivanja

Bazna klasa

  • Bazna klasa System nema matricu jer je Matrix apstraktna klasa. Ona stoga može predstavljati tzv. eksplicitni sistem u kome se riješenje dobiva bez potrebe za stvarnim rješavanjem sustava.

  • Bazna klasa nije apstraktna. Sve metode imaju implementaciju: na primjer get_matrix() vraća nul-pokazivač.

  • Klasa ima konstruktor, virtualni destruktor i resize() metodu koja dozvoljava da imamo defaultni konstruktor koji ne alocira memoriju.

  • Metode get_matrix, get_rhs, get_solution su standardne get metode, s time da smo kod get_matrix prisiljeni vratiti pokazivač (mogli bi i smart pointer) jer bazna klasa nema matricu.

  • Asembliranje sustava je zadatak korisnika sustava. Korisnik će klasi dati pokazivač na funkciju koja radi asembliranje, koju ćemo smjestiti u zaštićenu varijablu članicu: 

void (*user_assembly_fun)(System*);

  • Za to nam služi metoda članica attach_user_assembly_fun koja je kodirana na ovaj način: 

virtual void attach_user_assembly_fun(void (*f)(System*)){user_assembly_fun = f;}

Bazna klasa: asembliranje

  • Asembliranje radi funkcija assembly() koja samo delegira posao korisničkim funkcijama. Implementacija je ovdje: 

void System::assembly()

{
    if(user_assembly_fun != 0)
        user_assembly_fun(this);
    else
        user_assembly();
}

  • Objašnjenje: assembly() će pozvati funkciju user_assembly_fun() ako je korisnik inicijalizirao pokazivač na nju. Ukoliko nije poziva se funkcija članica user_assembly() koja u klasi System ima praznu implementaciju. Zašto te dvije mogućnosti?

  • user_assembly_fun() uzima pokazivač na System kroz koji može dohvatiti pokazivač na Matrix. To znači na korisnik može programirati asembliranje matrice sustava samo kroz sučelje Matrix i metode koje ono nudi. Na taj je način asembliranje matrice neovisno o tipu matrice. To u nekim slučajevima može biti nemoguće ili neefikasno. Stoga klasa nudi i drugu mogućnost:

  • Matrica se može asemblirati i kroz metodu članicu user_assembly(). Za to je potrebno napisati klasu koja proširuje klasu System (ili neku njenu podklasu) i reimplementirati metodu user_assembly(). Prednost tog postupka je u tome što user_assembly() može raditi sa specifičnom matricom, a ne samo sa sučeljem Matrix, te što u proširenu klasu možemo spremiti dodatne informacije potrebne za asembliranje matrice.

Klase FullSystem i BandSystem

  • Izvedene klase čuvaju matricu sustava u odabranoj formi. Stoga moraju implementirati konstruktore i resize() metodu. Uočimo da smo u BandSystem klasi morali ne samo preraditi resize() metodu već i dodati novu s dodatnim argumentima.

  • Ovim klasama dodajemo nove varijable članice potrebne za implementaciju solve() metode. U našem slučaju koristimo LAPACKove rutine pa nam stoga treba jedan vektor cijelih brojeva (p_ivec) koji pamti permutaciju redaka matrice.

  • Prerađujemo get_matrix() metodu u kojoj koristimo kovarijantan povratni tip kako bismo smanjili gubitak informacije.

  • Metodu user_assembly() ne prerađujemo. To ostavljamo izvedenim klasama.

  • Konačno, treba implementirati solve() metodu.

Rješavanje sustava korištenjem CLAPACK paketa

Za rješavanje linearnih sustava koristimo biblioteku LAPACK (www.netlib.org/lapack) koja je pisana u FORTRAN-u.

LAPACK je preveden u jezik C pomoću f2c konvertera i ta se biblioteka naziva CLAPACK (www.netlib.org/clapack/, vidi još clapack/FAQ). Kao takva može se koristiti u C++ programima.

Ako želimo koristiti funkciju pisanu u C-u u C++ programu (koji se kompilira s C++ kompajlerom) moramo deklaraciju funkcije uključiti unutar extern "C" deklaracije. Na primjer:

extern "C" {
    void dgesv_(integer* N, integer*  NRHS, double* A, integer* LDA, integer* IPIV,
                double* B, integer* LDB, integer* INFO);

    void dgbsv_(integer* N, integer* KL, integer* KU, integer* NRHS, double*  AB,
                integer* LDAB, integer* IPIV,  double* B, integer* LDB, integer* INFO);
}

  • dgesv_ riješava sustav s punom matricom (vidi www.netlib.org/lapack/double/dgesv.f).

  • dgbsv_ riješava sustav s trakastom matricom (vidi www.netlib.org/lapack/double/dgbsv.f).

Gornji linkovi objašnjavaju poziv FORTRANskih rutina i značenje argumenta. Kada se isti poziv vrši u programskom jeziku C (ili C++) onda:

  • Sve skalarne tipove treba zamijeniti s pokazivačima na odgovarajuće tipove;

  • Matricu treba zamijeniti s pokazivačem na elemente "spremljene po stupcima".

Detaljnije informacije potražiti na web-u.

Primjena

Treba napraviti System-klasu za rješavanje rubnog problema za običnu diferencijalnu jednadžbu metodom konačnih diferencija. Zadaća je sljedeća:

-u'' + b u = f(x)  za x iz (0,1),
u(0) = u(1) = 0,

gdje je b > 0 konstanta, a f(x) zadana funkcija.

Implementacija

Upotreba različitih matričnih formata

int main(int argc, char** argv)
{
    if(argc < 3) throw std::invalid_argument("Expecting arguments: n system_type");
    std::stringstream  ss(argv[1]);

    double b=1.0;

    integer n;
    ss >> n;

    double h = 1.0/(n+1);
    std::string stype(argv[2]);

    System* psys = 0;
    if(stype =="Full")
    {
       psys = new FullSystem();
       psys->attach_user_assembly_fun(poisson);
       psys->resize(n);
    }
    else if(stype =="Band")
    {
       psys = new BandSystem(n,1,1);
       psys->attach_user_assembly_fun(poisson);
    }
    else if(stype =="ODJBand"){
       psys = new ODJBandSystem(n,b);
    }

    psys->assembly();

    std::cout << " Matrix: ======" << std::endl;
    std::cout << *psys->get_matrix();
    std::cout << " RHS   : ======" << std::endl;
    std::cout << psys->get_rhs();

    psys->solve();

    std::cout << " Solution: ====" << std::endl;
    std::cout << psys->get_solution();

    Vector exact_solution(n);
    for(int i=0; i < n; ++i) exact_solution(i) = std::sin(M_PI*(i+1)*h);

    exact_solution -= psys->get_solution();
    exact_solution *= h;
    std::cout << " Greska u rješenju = " << exact_solution.norm() << std::endl;

    delete psys;
    return 0;
}

Zadatak

  • Implementirati sve navedene klase.

  • Instalirati CLAPACK i povezati System-klase s njime tako da ispravno pozivaju rutine dgesv_ i dgbsv_.

  • Testirati implementaciju rješavanjem postavljene rubne zadaće na primjerima gdje se zna točno rješenje. Mjeriti grešku aproksimativnog rješenja.

  • Formirajte rubnu zadaću koristeći user_assembly_fun i user_assembly (preoterećene u ODJBandSystem). Koje su prednosti/nedostaci pojedinog pristupa?
